# Glossary

This glossary defines technical terms, acronyms, and domain‑specific concepts used in the CMS Platform documentation. Use it as a quick reference when reading other sections.

---

## A

**Access Token** – A short‑lived JWT (30 minutes) used to authenticate API requests. Stored only in RAM on the frontend to prevent XSS attacks.

**Admin** – A user role with elevated permissions for moderation, user management, and system configuration.

**Alembic** – A database migration tool for SQLAlchemy. Used to manage schema changes across environments.

**Argon2** – A modern password‑hashing algorithm used by the backend to securely store user passwords.

**Audit Log** – A record of administrative actions stored in the `admin_audit_logs` table for compliance and tracking.

**Author** – A user role with permissions to create, edit, and publish content (posts).

**Axios** – A promise‑based HTTP client used in the frontend for API communication with automatic token refresh.

**Azure Blob Storage** – Microsoft's cloud object storage solution. Used to store uploaded images, videos, and profile pictures.

---

## B

**Block** – The smallest unit of content in a post. Can be a `text`, `image`, or `video` block. Each block has a `type` and `data`.

**Builder Canvas** – The main editor interface where users drag and drop sections, columns, and blocks to build a post.

---

## C

**Category** – A taxonomy term for grouping posts (e.g., "Technology", "Lifestyle"). Admins manage categories.

**CDN** – Content Delivery Network. Azure Blob Storage can be integrated with a CDN for faster global asset delivery.

**CMS** – Content Management System. The core product.

**Column** – A structural container within a section. Each column holds a list of blocks.

**Comment** – User‑generated text on a published post.

**Context API** – React's built‑in state management solution. Used for Auth, Notifications, and Theme.

**Cookie** – Small piece of data stored by the browser. Refresh tokens are stored in HttpOnly, Secure, SameSite cookies.

**CORS** – Cross‑Origin Resource Sharing. Configured in FastAPI to allow only trusted origins.

---

## D

**dnd‑kit** – A lightweight drag‑and‑drop library used in the frontend for reordering sections, columns, and blocks.

**Draft** – An unpublished post with `status = "draft"`. Only the author can edit and view drafts.

---

## E

**Embedding** – A numerical vector representation of text, generated by a model like `all-MiniLM-L6-v2`. Used for semantic search in RAG.

**Endpoint** – A specific URL path and HTTP method that exposes a backend API.

**Enum** – A set of named constants (e.g., `UserRole`, `ViolationType`). Used in models and schemas.

**Eviction** – The process of forcibly closing all WebSocket connections for a user, typically during suspension.

---

## F

**FAISS** – Facebook AI Similarity Search. A library for efficient similarity search and clustering of dense vectors. Used in the RAG pipeline.

**FastAPI** – The Python web framework used for the backend. Provides high performance, async support, and automatic OpenAPI documentation.

**Feed** – A paginated, sorted list of published posts visible to all authenticated users.

**Fixed Window** – A rate‑limiting algorithm that counts requests within a fixed time window (e.g., 5 minutes). Used with Redis.

---

## G

**Google OAuth** – An authentication method that allows users to log in using their Google account.

**Groq** – The LLM provider used for summarization and Q&A. Model: `llama-3.1-8b-instant`.

**Guest Route** – A route that only unauthenticated users can access (e.g., login, register). Authenticated users are redirected away.

---

## H

**HttpOnly** – A cookie flag that prevents JavaScript access, protecting against XSS attacks.

**Hard Delete** – Permanent removal of a record from the database (e.g., `DELETE /admin/posts/{id}/hard-delete`).

---

## I

**Index** – A database structure that speeds up query performance. Many tables have indexes on foreign keys and frequently filtered columns.

**Interaction** – User actions on posts: liking (`PostLike`) and commenting (`Comment`).

---

## J

**JSONB** – A PostgreSQL data type that stores JSON in a binary format. Used for `post.content` and `template.schema`.

**JWT** – JSON Web Token. Used for authentication (access and refresh tokens). Signed with HS256.

---

## L

**Like** – A user's expression of appreciation for a post. Stored in `post_likes`.

**L2 Distance** – Euclidean distance used by FAISS for similarity search.

**localStorage** – Browser storage. Used for theme preference and auto‑saving post drafts (not for tokens).

---

## M

**MDX** – Markdown with JSX. The file format used for Docusaurus documentation.

**Media** – Images and videos uploaded by users. Stored in Azure Blob and referenced via the `media` table.

**Middleware** – Code that runs before request handling. The AuthMiddleware validates JWT and attaches the user.

**MUI** – Material UI, the component library used for the frontend.

---

## N

**Notification** – A real‑time or historical message sent to a user (e.g., like, comment, admin action).

**NotificationService** – Backend service that creates notifications and pushes them via WebSocket.

---

## O

**ORM** – Object‑Relational Mapping. SQLAlchemy is the ORM used to map Python classes to database tables.

**OTP** – One‑Time Password. A 6‑digit code sent via email for verification (registration, password reset). Stored in Redis with a TTL.

---

## P

**Post** – The primary content entity. Contains a `title`, `content` (JSONB), `status`, and metadata.

**PostBuilder** – The visual drag‑and‑drop editor used to create and edit posts.

**Profile** – A user's public information: `name`, `avatar_id`, and `avatar_url`. One‑to‑one with `User`.

**ProtectedRoute** – A frontend component that guards routes based on authentication and role.

**Published** – A post with `status = "published"`. Visible in the feed and accessible via the public API.

---

## R

**RAG** – Retrieval‑Augmented Generation. The AI pipeline that combines vector search with LLM generation for summarization and Q&A.

**Rate Limiting** – A mechanism that restricts the number of requests a user can make in a given time window. Implemented using Redis.

**Redis** – An in‑memory data store used for caching, OTP storage, refresh tokens, and rate limiting.

**Refresh Token** – A long‑lived token (7 days) stored in an HttpOnly cookie. Used to obtain new access tokens.

**Report** – A user‑submitted flag for content that violates community guidelines.

**Repository** – A data‑access layer that encapsulates SQLAlchemy operations. Used by services to interact with the database.

**Router** – In FastAPI, a grouping of related endpoints (e.g., `posts.router`).

**Role** – A user's permission level: `admin` or `author`.

---

## S

**SAS Token** – Shared Access Signature. Not used currently; Azure blobs are public.

**Schema** – A Pydantic model for request/response validation, or a template's allowed structure.

**Section** – A top‑level container in a post. Each section contains one or more columns.

**SentenceTransformers** – A Python library for generating sentence embeddings. Used with the `all-MiniLM-L6-v2` model.

**Service** – A business‑logic layer. Services orchestrate repositories, caching, notifications, and external APIs.

**Soft Delete** – Marking a record as deleted without removing it from the database (`is_deleted = True`).

**SQLAlchemy** – The Python SQL toolkit and ORM used to interact with PostgreSQL.

**Supabase** – The PostgreSQL hosting provider used in production.

**Suspension** – Temporarily blocking a user from accessing the system. Set by admins with an expiration time.

---

## T

**Tag** – A free‑form taxonomy term that can be attached to posts. Authors can create tags on the fly.

**Template** – A predefined structure for posts, defining allowed block types and limits (max sections, columns, blocks).

**TipTap** – A rich‑text editor library used for `text` blocks in the post builder.

**TOC** – Table of Contents. Automatically generated on each documentation page.

**TTL** – Time‑To‑Live. The expiry time for a Redis key (e.g., OTP: 5 minutes).

---

## U

**UUID** – Universally Unique Identifier. Used as primary keys for all tables.

---

## V

**VectorStore** – A wrapper around FAISS that handles embedding generation, indexing, and search.

**Violation Type** – A category for content violations (e.g., `HATE_SPEECH`, `SPAM`). Used in reports and audit logs.

**Vite** – The frontend build tool used for fast development and production builds.

---

## W

**WebSocket** – A persistent, bidirectional communication channel used for real‑time notifications.

**Window** – In rate limiting, the time interval over which requests are counted (e.g., 5 minutes).

---

## X

**XSS** – Cross‑Site Scripting. A security vulnerability mitigated by storing access tokens in RAM and refresh tokens in HttpOnly cookies.

---

## Y

**Yup** – A JavaScript schema validation library used with Formik for form validation.

---

## Z

**Zod** – (Not used; Yup is used instead.)

---
